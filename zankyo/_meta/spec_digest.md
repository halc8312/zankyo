# 仕様書ダイジェスト

## 00_role_and_principles.md 要点
- 本企画の目的は、単発の設定メモではなく、小説・TRPG・ゲーム・設定集の母体として運用できる設定資料群を構築すること。
- 歴史・民俗・宗教・言語・批評を一体で扱い、世界を「思いつき」でなく「検証済み構造」として作る。
- 抽象語や神秘語で逃げず、匂い、労働、制度、食、葬制、偏見、身体感覚まで落とし込む。
- AI臭の強い固有名詞、既存ファンタジーの定型、善悪二元論、魔力万能論、均質な説明口調を禁じる。
- 地域差、記録矛盾、沈黙、偏見、土着の不格好さを意図的に残し、説明し尽くさない。
- 各ファイルに諺や祈り、法令、歌、証言など人間の声を混ぜ、文書形式も分ける。
- 歴史・残響倫理・地方文化・社会制度・言語・人物を優先的に掘り下げる。

## 01_world_core.md 要点
- 異能は個人才能ではなく、土地に染みついた過去の出来事の残響である。
- 発現条件は居住歴であり、血統は本質条件ではない。血統に見える現象も制度と土地支配で説明する。
- 土地を離れると残響は薄れ、土地ごとに系統は完全に異なる。
- 強い残響はたいてい悲劇の現場に宿り、恩恵と代償を同時にもたらす。
- 渡りは複数の土地を渡り歩きながら残響をなぞる者たちで、世界倫理の焦点となる。
- 国家や共同体が意図的に悲劇を起こし、残響を資源化する可能性が最大の倫理問題である。
- 主題は故郷・継承・忘却・悲劇の商品化・公式史と証言の対立に通底させる。

## 02_meta_files.md 要点
- 生成前に `_meta/` 配下の統制ファイルを必ず整え、全Phaseで再参照する。
- workplan、style_bible、naming_rules、id_registry、consistency_matrix、search_log、manifest、quality_rubric、assumptions が必須。
- ID は時代、事件、戦争、国家、地方、都市、宗教、言語、残響、派閥、人物、禁忌、研究ノートで体系化する。
- 人物は最低1つの事件・地方・派閥・残響に紐づける。
- 都市は最低1つの事件と残響に紐づける。
- 宗教・法・経済は残響制度と必ず接続する。
- 意図的な記録矛盾は `異説:` として明示し、意図しない矛盾は禁止される。

## 03_density_and_templates.md 要点
- 各ファイルは概要、詳細、生活への影響、歴史的起源、ID接続、異説、物語フック、参照元を含む構成を基本とする。
- 研究ノート800字以上、理論・言語・地方などは1500字以上など、明確な最低密度が定められている。
- 1ファイル8000字上限、300字下限を守り、分割時は必ず親インデックスを置く。
- リサーチノートには要約、重要論点、転用構造、借用回避の注意、URL、反映先を必ず記録する。
- 学術機関・博物館・研究機関・公的ソースを優先し、Wikipedia 単独依存は禁止。
- 同一トピックの検索は三回までとし、それ以上は切り口を変える。
- 採用しなかった検索も理由とともにログに残す。

## 04_phase_0_to_2.md 要点
- Phase 0 は最低20本の研究ノートと、借用防止・クリシェ排除の補助文書を作る。
- トラウマと土地、埋葬、戦跡、ディアスポラ、非西欧前近代社会、記憶研究、言語学、既存ファンタジー差別化などを調べる。
- Phase 1 では中核世界観、地理概観、宇宙観、残響理論、禁忌、暦、尺度、基礎語彙、季節性を作る。
- resonance_theory には居住年数、年齢差、干渉、飽和、逆残響、依存、医療・宗教・法・軍事を含める。
- 数値モデルとして強度ランク、居住年数閾値、距離減衰、時間減衰、季節係数、人口密度関係、相性表を定義する。
- Phase 2 は最重要で、150件以上の公式編年、60件以上の民間史、50件以上の敗者史、30件以上の異端史、3空白時代を要する。
- 人為的悲劇を最低5件組み込み、公式史と敗者史の双方に異なる記述で現す。

## 05_phase_3_to_5.md 要点
- Phase 3 では現在の地理と勢力を整理し、12地方以上と25都市以上を具体的に描く。
- 地方ごとに気候、水系、匂い、病、労働、食、法、偏見、異様さ、歴史的傷、庶民生活スケッチを要する。
- 都市ごとに成立史、区画、水衛生、夜の灯り、市場、支配層、危険地帯、噂、匂い、音を記す。
- Phase 4 は最低8勢力・6宗教・5言語を含み、法・家族・教育・刑罰・葬制・残響国家方針まで記述する。
- 言語設計では音韻・音節・禁則・語順・敬語・罵倒・祈祷語・地名人名形成・80語語彙・10例文・文字体系が必須。
- Phase 5 では残響カテゴリー、50件以上の個別残響、腐食と代償、禁忌実践、法・医療・経済を整理する。
- 渡り専用サブディレクトリを設け、隠語60語以上、巡回路5本以上、著名渡り10名以上など文化集団として扱う。

## 06_phase_6_to_9.md 要点
- Phase 6 では15派閥以上と7つの現行紛争、影の権力、核心的倫理問題を整理する。
- 派閥は目的、資金源、建前、本音、敵味方、歴史観、越えた一線、越えない一線、内部対立まで示す。
- Phase 7 は主要人物25名以上、脇役60名以上、Mermaid関係図を要する。
- 主要人物には声、癖、秘密、恥、宝物、嘘、会話3行、仕草3種、沈黙、今朝の食事、最後に泣いた時点が必須。
- 各人物は地域の音韻・敬称・罵倒語彙を反映した話し方を持つべきである。
- Phase 8 では食、諺、詩歌、都市伝説、葬制、子供、身振り、罵り、労働歌、季節不安など生活テクスチャを集積する。
- Phase 9 では長編候補5本、サブプロット25本、謎15本、主題整理を行い、歴史と残響倫理が現代化する構造にする。

## 07_phase_10_to_11.md 要点
- Phase 10 は最低3周の自己批評ループを実施し、毎回問題列挙と修正を行う。
- differentiation、ai_smell_audit、consistency_audit、naming_audit、orphan_reference_audit、scorecard など監査ファイルが必須である。
- scorecard は独自性・土着性・整合性・感覚性・倫理的緊張など10項目を0〜3点評価し、2点以下を修正する。
- Phase 11 では README、GLOSSARY、MAP、TIMELINE_MASTER、ID索引、読書導線、利用ガイド、要約版を作る。
- エントリー用として導入散文、1枚世界紹介、3種ピッチ、12地方場面集を別ディレクトリに配置する。
- README 冒頭には読むべき5ファイルを掲げ、全ファイルへ到達可能にする。
- インデックス群は100超ファイルの迷子防止装置として機能させる必要がある。

## 08_execution_strategy.md 要点
- 実行は骨格確定、肉付け、検証の三段で進める。
- 骨格段階では `_meta/`、音韻ルール、時代区分、150事件タイトル、12地方骨格、ID初期化だけを先に確定し、整合するまで詳細化しない。
- 肉付けは歴史→残響→地理→社会→人物→政治→生活→物語の順に行う。
- Phase 単位で最低12コミットに分け、巨大1コミットを禁ずる。
- Web検索40回以上、ファイル120以上、総文字数20万以上、全体読み返し必須。
- 部分提出時は優先度に従い、未達理由を `INCOMPLETE.md` に記録する。
- PR 説明には深さ3のツリー、各Phaseサマリ、総ファイル数、総文字数、検索回数、差別化ポイント等を必ず含める。

## 09_kpi_and_dod.md 要点
- 戦争以外の歴史イベント30%以上、強力残響の非戦闘由来15%以上など構成比KPIがある。
- 主要人物のうち渡り最低5名、国家以外の主要勢力最低3つ、宗教の対立軸、言語間の支配/差別語彙関係も必須である。
- 用語集300項目中50項目以上は既存ファンタジー語の代替独自語を含む。
- 主要人物25名全員、地方12全てが事件・残響・傷に結びつく必要がある。
- DoD は0〜11全ファイル存在、120ファイル以上、20万字以上、検索40回以上、研究20本以上など数値要件で構成される。
- AI臭監査0件、命名逸脱重大0件、孤立参照0件、READMEから全ファイル到達可能が終了条件である。
- PR 説明文にも達成チェックリストの記載が求められる。

## 10_internal_checks.md 要点
- 開始前に、能力世界ではなく歴史世界になっているか、暮らしや葬りまで見えているかを自問する。
- 強い土地の倫理矛盾が国家・宗教・家族・個人へ波及しているかを確認する。
- 公式史・民間伝承・敗者証言の語り分け、音韻準拠、物語利用性、湿った不格好さを確認する。
- 骨格確定を飛ばさない、ID確定前に固有名詞を乱造しない、検索貼り付けで済まさないことが追加の開始前チェックである。
- 人為的悲劇は具体事件として書く必要があり、抽象論のままでは不可。
- 渡りは小道具ではなく文化集団として扱う。
- 各Phase終了時には必須ファイル、密度、ID整合、命名準拠、文体差別化、抽象語過多を再点検する。

## 最重要禁則・KPI 上位5
1. AI臭の強い固有名詞、既存ファンタジー定型、抽象語逃げを禁止する。
2. Definition of Done を100%満たすまで停止しない。
3. 骨格確定段階を終える前に肉付けへ進まない。
4. 主要構成要素はすべてIDで接続し、孤立参照を残さない。
5. 歴史偏重ではなく歴史厚みを出しつつ、生活・制度・残響倫理・地方差を必ず現場感覚で描く。

## 気づいた矛盾・不明瞭点
- 問題文は `.copilot-spec/` を参照しているが、実在パスは `copilot-spec/` である。以後は実在パスを採用する。
- `manifest.md` は仕様上 `_meta/` 配下だが、問題文後半では `_meta/manifest.md` と裸名 `manifest.md` が混在する。 `_meta/manifest.md` を正本とし、README から誘導する。
- `08_execution_strategy.md` は骨格段階のコミットを要求するが、同時に Phase 0 の大規模調査も必須としている。先に骨格メタを作り、その後 Phase 0 調査を走らせる解釈を採用する。
- 想定読了時間の算出方法は未指定のため、最終PRでは総文字数から概算を置く。
