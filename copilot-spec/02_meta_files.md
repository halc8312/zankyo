# 02. メタファイルとID規則

## 生成前に必ず作るメタファイル → `zankyo/_meta/`
以下を先に作成し、その後の全Phaseで参照せよ。

- `workplan.md`: 作業順序、優先度、依存関係
- `style_bible.md`: 文体、禁則、描写原則、抽象語禁止リスト、質感指針
- `naming_rules.md`: 地域別の音韻ルール、語形成、敬称、地名接尾辞、蔑称、古語層
- `id_registry.md`: すべてのID体系
- `consistency_matrix.md`: 地域・言語・派閥・宗教・残響・時代・主要事件の対応表
- `search_log.md`: Web検索ログ（検索語、目的、取得先、何に反映したか）
- `manifest.md`: 生成ファイル一覧、概算文字数、更新履歴
- `quality_rubric.md`: 独自性・土着性・整合性・感覚性・倫理的緊張の評価基準
- `assumptions.md`: 不可避に置いた仮定とその理由

## 共通ID規則（必須）
世界全体の整合性維持のため、以下のIDを導入せよ。

- 時代: `ERA-01`
- 事件: `EVT-001`
- 戦争: `WAR-01`
- 王朝・国家: `POL-01`
- 地方: `REG-01`
- 都市: `CITY-01`
- 宗教: `REL-01`
- 言語: `LNG-01`
- 残響: `RES-001`
- 派閥: `FAC-01`
- 人物: `CHR-001`
- 禁忌: `TAB-01`
- 研究ノート: `R-01`

## 相互参照ルール
- 人物は最低1つの `EVT`、1つの `REG`、1つの `FAC`、1つの `RES` に紐づくこと
- 都市は最低1つの `EVT` と 1つの `RES` に紐づくこと
- 宗教・法・経済は必ず残響制度と接続すること
- 意図的な記録矛盾は `異説:` セクションで明記すること
- 意図しない矛盾は禁止
