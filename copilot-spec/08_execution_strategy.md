# 08. 実行戦略

## 二段階生成原則
本タスクは長大であるため、以下の二段階で実行せよ。粗密のムラを避けるため。

### 第1段（骨格確定段階）
以下を先に確定し、コミットせよ：

1. `_meta/` 全ファイル
2. `naming_rules.md` の音韻ルール（地域別5系統以上）
3. `eras.md` の時代区分（名称と年代だけ先に固定）
4. `chronicle_official.md` の事件タイトルと年代だけ先に150件リスト化
5. `regions.md` の12地方の名称・位置・気候のみ先に確定
6. `id_registry.md` を全 ID 体系で初期化

これらが互いに整合するまで肉付けに進むな。

### 第2段（肉付け段階）
骨格に従って各ファイルを詳細化。Phase 2 → 5 → 3 → 4 → 7 → 6 → 8 → 9 の順で進めよ。

理由: 歴史 → 残響 → 地理 → 社会 → 人物 → 政治 → 生活 → 物語 の順で因果が下流に流れるため。

### 第3段（検証段階）
Phase 10 を最低3周。

## コミット戦略
- Phase 単位でコミットを分けよ（最低12コミット）
- 各コミットメッセージに「Phase X: 主要追加項目」を明記
- 巨大1コミット禁止

## サブエージェント/ツール活用の必須要件
- Web検索ツール 最低40回
- ファイル生成 最低120ファイル
- 総文字数 20万字以上
- 並列可能なら並列化
- 生成後、必ず全体読み返しを行う
- 検索結果は `search_log.md` に記録
- 研究結果がどの設定に反映されたかを明記

## フォールバック規定
タイムアウト・ツール上限などで全Phase完走が困難になった場合、以下の優先度で部分提出せよ。

### 最優先（必達）
- `_meta/` 全ファイル
- Phase 1 (core)
- Phase 2-A (chronicle_official, chronicle_defeated, eras)
- Phase 2-C (resonance_map)
- Phase 3 (regions, cities)
- Phase 5 (resonance_categories, notable_resonances 最低30)
- Phase 11 (README, GLOSSARY, CANON_SUMMARY)

### 次点
- Phase 4, 7, 10

### 後回し可
- Phase 8 の追加詳細
- Phase 9 の subplots
- Phase 6 の詳細

部分提出時は `INCOMPLETE.md` にて未達項目と理由、再開時の指針を残せ。

## レビュー可能性確保
- PR の説明文冒頭にディレクトリツリー(深さ3まで)を貼ること
- ファイル数100超なので、`README.md` の冒頭に「とりあえず読むべき5ファイル」を明示せよ
- Phase 単位でコミットメッセージを切ること（前述）

## PR提出フォーマット
タイトル: `zankyo: 残響の大陸 完全設定資料（歴史強化版）`

本文に必ず含めること:
- ディレクトリツリー（深さ3）
- 各Phase達成サマリ
- 総ファイル数
- 総文字数
- 検索回数
- 想定読了時間
- 自信のある章 Top 5
- 意図的に未解明として残した謎
- 差別化ポイント要約
- 自己批評で発見し修正した代表的問題
- 達成チェックリスト
