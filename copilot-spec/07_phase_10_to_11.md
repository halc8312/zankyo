# 07. Phase 10〜11

## Phase 10: 自己批評ループ（最低3周） → `zankyo/10_review/`
最低3ラウンド実施せよ。

1. 全ファイル読み返し
2. `round_N_issues.md` に問題列挙
3. 修正
4. 辛口批評家視点で再点検
5. 歴史考証マニア視点で再点検
6. AI臭監査
7. 差別化再確認

### 必須ファイル
- `round_1_issues.md`
- `round_2_issues.md`
- `round_3_issues.md`
- `differentiation.md`
- `ai_smell_audit.md`
- `consistency_audit.md`
- `naming_audit.md`
- `orphan_reference_audit.md`
- `scorecard.md`
- `scorecard_history.md`

### 監査項目
- 禁則事項違反
- 音韻ルール逸脱
- 記録矛盾の未明示
- 文化の見た目だけ借用
- 戦争偏重
- 土地と能力の接続不足
- 人物と歴史の接続不足
- 抽象形容詞の多用
- 説明文の文体単調化
- 「世界観のための世界観」になっており、生活が見えない箇所

### 検証スコアカード
`scorecard.md` を作成し、以下の各項目を 0–3点 で自己採点せよ。2点以下の項目はすべて修正すること。

| 項目 | 採点基準 |
|------|---------|
| 独自性 | 既存ファンタジーの類型に陥っていないか |
| 土着性 | 風土・労働・身体感覚が見えるか |
| 整合性 | ID参照・年代・因果の矛盾がないか |
| 感覚性 | 匂い・湿度・音が描かれているか |
| 倫理的緊張 | 善悪二元論を避け、立場の衝突になっているか |
| 歴史厚み | 公式史と他の歴史の食い違いが機能しているか |
| 残響密度 | 各地方・人物・事件が残響に紐づいているか |
| 言語精度 | 命名が音韻ルールに従っているか |
| 文体多様性 | 文書タイプごとに文体が変わっているか |
| 物語接続 | 設定が物語として使える形になっているか |

採点根拠と修正履歴を `scorecard_history.md` に残せ。
